# kmp-architecture-guard

## 목적
KMP 프로젝트의 아키텍처 규칙 위반을 조기에 탐지한다.

## 체크 포인트
- `shared`에서 플랫폼 전용 API 직접 참조 여부
- `shared` DTO와 `server` 계약 불일치 가능성
- 모듈 의존 방향 역전 여부
- 설정 파일 변경 시 정책 문서 반영 누락

## 권장 실행 시점
- PR 생성 전
- 공통 모델/계약 수정 직후
- 모듈 구조 리팩토링 직후

## 출력 형식
- Severity(P0-P3)
- 영향 모듈
- 재현 조건
- 수정 제안(짧고 실행 가능하게)
